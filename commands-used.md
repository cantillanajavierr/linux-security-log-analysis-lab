This project analyzes Linux authentication logs to identify suspicious activity such as failed authentication attempts and privileged command execution.

- grep

1. Reviewed log files in /var/log

3. Searched for authentication failure events
4. Counted failed authentication attempts
5. Investigated sudo activity
6. Created an automation script to analyze logs

# Commands Used

View log directory
ls /var/log

Identify authentication log
ls /var/log/auth.log /var/log/secure 2>/dev/null

Search authentication related failures
sudo grep -Ei "failed|failure|authentication|pam_unix|su:" /var/log/auth.log

Count authentication failures
sudo grep -Ei "failed|failure|authentication" /var/log/auth.log | wc -l

Review sudo activity
sudo grep "sudo" /var/log/auth.log | tail -n 20

Run automation script
sudo ./scripts/detect_auth_activity.sh

Save script output
sudo ./scripts/detect_auth_activity.sh > notes/script-results.txt
EOF

# Security Findings

## Summary
This investigation analyzed Linux authentication logs to identify authentication failures and privileged command execution.

## Findings

Authentication failure events were detected in the logs.

These failures were generated by unsuccessful attempts to switch users using the su command.

The system logs also recorded sudo commands executed by the user.

## Risk

Repeated authentication failures could indicate unauthorized access attempts or incorrect credential usage.

## Recommendations

Review authentication failures regularly.

Monitor privileged command usage.

Restrict unnecessary sudo privileges.

Consider implementing additional protections such as fail2ban for systems exposed to SSH.
